Get Flutter Games running on your machine in just a few minutes.

## Prerequisites

Before you begin, ensure you have:

- Flutter SDK (3.10.0 or higher)
- Dart SDK (3.0.0 or higher)
- A code editor (VS Code, Android Studio, or IntelliJ IDEA)
- A device or emulator to run the app

<Steps>
  <Step title="Clone the repository">
    Clone the Flutter Games repository to your local machine:

    ```bash
    git clone https://github.com/rayliverified/FlutterGames.git
    cd FlutterGames
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required dependencies using Flutter:

    ```bash
    flutter pub get
    ```

    This will install:
    - `cupertino_icons` (^1.0.8) - iOS style icons
    - `responsive_framework` (^1.5.1) - Responsive design utilities
    - `google_fonts` (^6.3.1) - Google Fonts integration
  </Step>

  <Step title="Run the app">
    Launch the app on your connected device or emulator:

    <CodeGroup>
    ```bash Debug Mode
    flutter run
    ```

    ```bash Release Mode
    flutter run --release
    ```

    ```bash Specific Device
    flutter run -d <device_id>
    ```
    </CodeGroup>

    The app will start and you'll see the home page with three sections: Games, Coming Soon, and Movies.
  </Step>
</Steps>

## Development Tips

<AccordionGroup>
  <Accordion title="Hot Reload">
    Press `r` in the terminal or save your file to hot reload changes instantly without losing app state.
  </Accordion>

  <Accordion title="Hot Restart">
    Press `R` in the terminal to perform a hot restart, which rebuilds the app from scratch while keeping it running.
  </Accordion>

  <Accordion title="Debug Mode Features">
    - Performance overlay: Shows FPS and frame rendering time
    - Widget inspector: Inspect the widget tree in VS Code or Android Studio
    - Debug banner: Indicates you're running in debug mode
  </Accordion>

  <Accordion title="Running on Multiple Platforms">
    Flutter Games supports multiple platforms:
    - **Android**: `flutter run -d android`
    - **iOS**: `flutter run -d ios` (macOS only)
    - **Web**: `flutter run -d chrome`
    - **Desktop**: `flutter run -d windows/macos/linux`
  </Accordion>
</AccordionGroup>

## Troubleshooting

<Warning>
If you encounter "No devices found", make sure you have:
- An emulator running, or
- A physical device connected with USB debugging enabled, or
- Chrome installed (for web development)
</Warning>

<Tip>
Run `flutter doctor` to check your Flutter installation and see if any components need to be installed or configured.
</Tip>

## Example App Flow

Here's the main app entry point from `lib/main.dart`:

```dart
import 'package:flutter/material.dart';
import 'package:flutter_games/pages/home.dart';
import 'package:flutter_games/themes.dart';

void main() {
  runApp(const FlutterGames());
}

class FlutterGames extends StatelessWidget {
  const FlutterGames({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Flutter Games',
      theme: lightTheme,
      darkTheme: darkTheme,
      home: const HomePage(),
      debugShowCheckedModeBanner: false,
    );
  }
}
```

The app initializes with a MaterialApp that:
- Sets up light and dark themes using Google Fonts
- Starts at the HomePage with bottom navigation
- Removes the debug banner for a clean UI
