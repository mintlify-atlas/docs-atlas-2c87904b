Flutter Games implements responsive design principles to ensure the app looks great on phones, tablets, and different screen sizes.

## Overview

Responsive design in Flutter involves:
- Adapting layouts to different screen sizes
- Using flexible widgets and constraints
- Implementing responsive typography and spacing
- Supporting different device orientations

## Fixed Aspect Ratios

### GameBoxItem

The game box maintains a consistent aspect ratio (`lib/ui/item_game_box.dart:5-29`):

```dart
class GameBoxItem extends StatelessWidget {
  static const IMAGE_RATIO = 1.50;

  GameBoxItem(this.buildContext, this.game, {this.width = 120});
  final BuildContext buildContext;
  final Game game;
  final double width;

  @override
  Widget build(BuildContext context) {
    double height = IMAGE_RATIO * width;

    return Material(
      borderRadius: BorderRadius.circular(4),
      clipBehavior: Clip.hardEdge,
      elevation: 8,
      shadowColor: Color(0xCC000000),
      child: FadeInImage.assetNetwork(
        image: game.box,
        placeholder: "assets/placeholder_box.png",
        width: width,
        height: height,
        fit: BoxFit.cover,
      ),
    );
  }
}
```

### Aspect Ratio Benefits

- **Consistent Proportions**: All game boxes have the same shape
- **Scalable**: Works at any size (120px in list, 150px in details)
- **Predictable Layout**: Prevents layout shifts

<Tip>
Define aspect ratios as constants to ensure consistency across your app and make it easy to adjust if needed.
</Tip>

## Best Practices

### 1. Use MediaQuery Sparingly

Only use MediaQuery when necessary:

```dart
// Good: For screen-width images
var screenWidth = MediaQuery.of(context).size.width;

// Better: Use flexible layouts
Expanded(child: ...)
Flexible(child: ...)
```

### 2. Prefer Flexible Widgets

Use Flutter's built-in responsive widgets:

- `Expanded` - Takes remaining space
- `Flexible` - Flexes within available space
- `ConstrainedBox` - Sets min/max constraints
- `FittedBox` - Scales content to fit

### 3. Set Explicit Constraints

Provide clear constraints to prevent layout errors:

```dart
SizedBox.fromSize(
  size: const Size.fromHeight(240),
  child: ListView.builder(...),
)
```

### 4. Handle Text Overflow

Always specify overflow behavior:

```dart
Text(
  game.name,
  maxLines: 1,
  overflow: TextOverflow.ellipsis,
)
```

### 5. Test on Multiple Devices

Test your responsive design on:
- Different screen sizes (small, medium, large)
- Different orientations (portrait, landscape)
- Different platforms (iOS, Android)
- Different pixel densities

<Tip>
Use Flutter DevTools to test different screen sizes without physical devices.
</Tip>

## Common Responsive Patterns

### Pattern 1: Fixed + Flexible

```dart
Row(
  children: [
    Container(width: 150), // Fixed
    Expanded(child: ...), // Flexible
  ],
)
```

### Pattern 2: Full Width with Padding

```dart
Padding(
  padding: const EdgeInsets.symmetric(horizontal: 16),
  child: SizedBox(
    width: double.infinity,
    child: ...,
  ),
)
```

### Pattern 3: Aspect Ratio Maintenance

```dart
Widget build(BuildContext context) {
  double height = IMAGE_RATIO * width;
  return Container(width: width, height: height);
}
```

### Pattern 4: Constrained Content

```dart
ConstrainedBox(
  constraints: const BoxConstraints(maxWidth: 120),
  child: Text(...),
)
```
