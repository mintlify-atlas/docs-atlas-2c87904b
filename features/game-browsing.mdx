The game browsing feature provides an intuitive interface for users to discover and browse the game catalog.

## Overview

The GamesPage displays games in a scrollable list format, allowing users to browse the available game collection efficiently.

## Implementation

### Main Games Page

The GamesPage component is implemented in `lib/pages/games_page.dart`:

```dart
import 'package:flutter/material.dart';
import 'package:flutter_games/models/game.dart';
import 'package:flutter_games/widgets/game_card.dart';

class GamesPage extends StatelessWidget {
  const GamesPage({super.key});

  @override
  Widget build(BuildContext context) {
    final games = Game.toList();
    return Scaffold(
      appBar: AppBar(
        title: const Text('Games'),
      ),
      body: ListView.builder(
        itemCount: games.length,
        itemBuilder: (context, index) {
          return GameCard(game: games[index]);
        },
      ),
    );
  }
}
```

### Key Components

#### AppBar

The page includes a simple AppBar with the title "Games" to identify the current section.

#### ListView.builder

The page uses `ListView.builder` for efficient rendering of the game catalog:

- **Performance**: Only renders visible items
- **Memory Efficient**: Items are built on-demand
- **Smooth Scrolling**: Optimized for vertical scrolling
- **Dynamic Content**: Handles any number of games

<Note>
ListView.builder provides lazy loading, which is essential for handling potentially large game catalogs without performance degradation.
</Note>

### GameCard Widget

Each game in the list is rendered by the `GameCard` widget (`lib/widgets/game_card.dart`):

```dart
class GameCard extends StatelessWidget {
  final Game game;

  const GameCard({super.key, required this.game});

  @override
  Widget build(BuildContext context) {
    return Card(
      margin: const EdgeInsets.all(8.0),
      child: Column(
        children: [
          Image.asset(
            game.image,
            fit: BoxFit.cover,
          ),
          Padding(
            padding: const EdgeInsets.all(8.0),
            child: Text(
              game.title,
              style: Theme.of(context).textTheme.titleLarge,
            ),
          ),
        ],
      ),
    );
  }
}
```

The GameCard provides a clean Material Design card layout with:
- Game cover image
- Game title
- Consistent margins and padding

## Data Model

The Game model (`lib/models/game.dart`) defines the structure for the simpler implementation:

```dart
class Game {
  final String title;
  final String image;
  final double price;
  final double rating;
  final String description;
  final List<String> screenshots;

  Game({
    required this.title,
    required this.image,
    required this.price,
    required this.rating,
    required this.description,
    required this.screenshots,
  });

  static List<Game> toList() {
    return dummyGames.map((item) {
      return Game(
        title: item['title'] as String,
        image: item['image'] as String,
        price: item['price'] as double,
        rating: item['rating'] as double,
        description: item['description'] as String,
        screenshots: List<String>.from(item['screenshots'] as List),
      );
    }).toList();
  }
}
```

The `toList()` static method converts dummy data into a list of Game objects for display.

## Best Practices

### Material Design Cards

The GameCard uses Material Design Card widget for consistent styling:
- Automatic elevation and shadows
- Rounded corners
- Consistent margins

### Image Display

Game cover images use `Image.asset` with `BoxFit.cover` to:
- Fill the available space
- Maintain aspect ratio
- Crop excess content appropriately

### Typography

The implementation uses theme typography for consistent text styling:
- `textTheme.titleLarge` for game titles
- Automatic color and font family from theme

<Tip>
Using theme styles ensures your app maintains visual consistency and makes theming changes easier.
</Tip>

## Performance Considerations

1. **ListView.builder**: Only builds visible widgets, not the entire list upfront
2. **const constructors**: The `const GamesPage()` constructor reduces unnecessary rebuilds
3. **Asset images**: Local asset images load quickly and are cached automatically
4. **Lazy loading**: Games are only rendered as they scroll into view
